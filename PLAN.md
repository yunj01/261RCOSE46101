Mitigating Catastrophic Forgetting in Small Korean Reasoning Models"**

> "**작은 한국어 LLM(3B)에 영어로 reasoning 능력을 먼저 학습시킨 후 한국어로 언어 적응을 시키는 2단계 방법론. Catastrophic forgetting을 Replay buffer + Lower LR 조합으로 방어.**"

## 🏗️ Method (핵심 contribution)

### Setup E: Two-stage Distillation (Ours)
```
Stage 1 (영어로 reasoning 능력 구축):
  - Data: 영어 CoT (Gemini 생성) - 7,473
  - Epochs: 3
  - LR: 2e-4
  - Goal: 모델이 영어로 reasoning skeleton 학습

         ↓

Stage 2 (한국어 적응 + 영어 retention):
  - Data: 한국어 CoT 90% + 영어 CoT 10% (replay buffer)
  - Epochs: 2
  - LR: 5e-5 ⭐ (Stage 1의 1/4 — 가중치 급격 붕괴 방지)
  - Goal: 한국어로 자연스럽게 출력하면서 영어 능력 유지
```

### Forgetting 방지 이중 전략 ⭐
1. **Replay buffer 10%**: Stage 2에서 영어 데이터 일부 섞어 학습
2. **Lower LR (1/4)**: Stage 2 학습률 낮춰 Stage 1 weight 보호

## 📊 Data

### 통제 변인: 모든 CoT는 **Gemini 2.5 Pro 단일 teacher** 생성 ⭐

이유: Teacher 다르면 응답 스타일 차이로 변수 오염. **언어**만 단일 변수로 통제.

### Source
- **GSM8K (영어 question + answer):** 7,473 train + 1,319 test
- **GSM8K-Ko (한국어 question, kuotient):** parallel 7,473
  - 한국어 reasoning은 무시 — 우리가 Gemini로 새로 생성

### Generated by Gemini 2.5 Pro
- **English CoT:** 7,473 (영어 질문 → Gemini → 영어 자연어 reasoning)
- **Korean CoT:** 7,473 (한국어 질문 → Gemini → 한국어 자연어 reasoning)
- 총 ~14,946 calls
- 비용: ~$15 (Pro tier 사용, $300 크레딧 안)

### Validation
- Gemini 답이 gold answer와 일치하는 것만 학습 데이터로
- 통과율 ~85% 예상 → ~12,500 valid CoT

### Evaluation Benchmarks
- **HRM8K** (한국어 native + 번역, 6,000+)
- **GSM8K test** (영어, 1,319)
- **KMMLU** (보조, 한국어 학문 — secondary)

## 🔬 Analysis (Section 5)

### Quantitative
- 5 setup × 2 main benchmark = 10 결과 매트릭스
- Forgetting metric: Setup E의 영어 성능 유지율
- Cross-lingual gap: in-language vs cross-lingual 차이

### Qualitative (1~2페이지)
**대표 sample 10~20개 ablation**:
- B (영어 CoT) → 한국어 질문 답할 때 영어로 응답 (어색함)
- C (한국어 CoT) → 영어 질문 답할 때 한국어로 응답 (반대)
- D (Mix) → 두 언어 섞이거나 일관성 X
- **E (Ours)** → 입력 언어 따라 자연스럽게 응답, reasoning 정확

**Error categorization:**
- 단위/단어 오류 (원, 개, dollars)
- 수식 전개 오류
- 언어 섞임 (code-switching)
- Reasoning step 누락
